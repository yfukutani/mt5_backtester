# DAY_EA 開発バックログ — デイトレード戦略200案

**作成日:** 2026.07.05
**目的:** 既存ブック（H4/D1スイング）とSCA（M15セッションORBスキャルパー）に続く第3のEA
「**DAY_EA**」の開発。対象は**デイトレード＝日中完結（数十分〜数時間保有・日跨ぎなし）**で、
SCAより保有が長く既存ブックより短い中間領域。M30/H1を主時間足とする。

**差別化方針:** SCAのエッジ（アジアレンジ→ロンドンブレイク）と重複しないこと。
候補の主軸は (1)日中トレンドフォロー (2)NYセッション (3)VWAP/日次レベル基準 (4)マルチシンボル相対。

## SCA開発で確立した事前知識（優先度に反映済み）

| 知見 | DAY_EAへの含意 |
|---|---|
| ✅ 時間・セッションの設計だけが機能 | セッション特化・時間帯選択系の事前確率を高く |
| ✅ リバーサル型（調整の逆流）は高品質 | 「前セッションの逆へ本流」型を優遇 |
| ✅ 同時シグナル=強い / DD後=回復力 | 停止・制限系は原則低優先 |
| ❌ 逆張り単体・確認追加・パターン系・BE/トレール系は全滅 | 該当カテゴリは初期優先度を下げる（文脈が違うため全排除はしない） |
| ❌ 指数・BTC・銀にセッションエッジなし | 銘柄はまずGOLD+JPYクロス+ドル系メジャーで |
| 検証プロトコル | ログ事前分析→EA実装→IS/OOS両立ゲート→プラトー確認（backlog2準拠） |

**検証仕様:** M30/H1エントリーでも日中SL/TPを使うため**最終判定はevery_tick**。
一次スクリーニングはopen_prices可（SL/TP距離がバー幅より十分大きい構成のみ）。
Magic: **20262000番台**。開発は単発戦略EA（DAY_XX.mq5）→採用分をDAY_EAに統合。
