# マルチタイムフレーム合流フィルター（PullbackTrend v1.3）

**採用日:** 2026.07.02
**対象:** PullbackTrend USDJPY H4 / GBPJPY H4（既存の本番採用戦略への改善）
**性質:** 新規の無相関源ではなく、既存の主力エッジの質を上げる改善。

## 背景

月利向上のため新規20戦略候補を提案・検証したが、A群（新市場への横展開: SILVER/日経225/NASDAQ/原油/
コモディティ通貨）・B群（週足スイング）は全て不成立（[rejected_strategies.md](rejected_strategies.md)）。
候補#9「マルチタイムフレーム合流」（既存ロジックへのフィルター追加）で初めて改善を確認した。

## 仕組み

`PullbackTrend.mq5` に入力 `UseHigherTFFilter` / `HigherTF` / `HigherTF_MA` を追加。
上位足（既定D1）のMA200トレンド方向がH4のエントリー方向と一致する場合のみエントリーを許可する。
既定`UseHigherTFFilter=false`で既存動作と完全一致（後方互換）。

## 経済的合理性

- **GBPJPYのIS期間が全く変化しない**（+491のまま）のは偶然ではなく、IS期間中D1とH4のトレンド方向が
  常に一致していた（強い一方向の円安トレンド）ためフィルターが一度も発動しなかったことを示す。
  逆にOOS期間（レンジ・逆行が多い時期）でフィルターが「D1と逆行する質の悪いH4シグナル」を除外し改善した。
  これは恣意的な過学習ではなく、経済的に筋の通るフィルタリング。
- **GOLDで効果がない理由:** GOLD別戦略7案がすべて棄却された通り、GOLD自体の値動きは既に
  「短期・タイトストップの順張り」だけで十分機能している。D1との合流を要求すると、GOLDが本来
  拾えていたH4スケールの初動の一部を取りこぼすため中立〜微減になったと考えられる。

## ポートフォリオ合算での効果（2026.07.02実測）

現行10枠（死に枠AUDJPY除く／全期間open_prices／各100k／USDJPYは2016起点整列）を `mt5bt portfolio` で
MTF OFF/ONを厳密比較：

| 構成 | 純利益 | 最大DD% | リターン/DD | 分散効果 |
|---|---|---|---|---|
| MTF OFF（旧） | +480,154 | 4.06% | 11.42 | 76%減 |
| **MTF ON（現状）** | **+487,887** | **3.99%** | **11.77** | 75%減 |

純利益+1.6%・DD低下・リターン/DD改善と合算でも一貫して向上。単体で大きく改善したUSDJPY/GBPJPYが
10枠中2枠のため、合算での効果はGOLD/Carryが支配する分だけ希釈される（それでも純増・DD減）。

## 本番反映

- `configs/pullback_usdjpy_h4.yaml` / `configs/pullback_gbpjpy_h4.yaml`: `UseHigherTFFilter=true` を追加。
- `configs/pullback_gold_h4.yaml`: 変更なし（既定OFF）。
- `experts/PortfolioEA.mq5` / `experts/PortfolioEA_OANDA.mq5`: SLEEVE構造体に`useHigherTF`/`higherTF`/
  `higherTFMA`/`hHigherTrend`を追加、`ProcPullback`に合流判定を追加。PB USDJPY/GBPJPY枠のみ既定ON、
  GOLD枠は既定OFFのまま。単枠バックテストで標準EAとの完全一致を確認（USDJPY +60,616、GBPJPY +18,337、
  GOLD +170,055＝無変更）。

## every_tick品質保証（2026.07.03）

PBはSL/TP注文を持ちバー内約定でモデル差が出うるため、本番config（USDJPY・MTF ON・risk2%・
2021-2026）をevery_tickで再実測: **+60,598 / PF1.67 / DD7.90% / 71取引** ＝ open_pricesの
+60,616とほぼ完全一致（差-18円は約定ノイズ）。**MTFフィルターの改善はバー内約定込みでも完全に保持**。
