## ⑤ 異常・要対応事項（記録のみ・対応判断はユーザー）

**致命的問題（🚨要対応）: なし。** EA稼働・銘柄・ロット・Magic すべて設計どおり。

以下は**軽微な確認事項（⚠️）**：

1. **⚠️ XAUUSD 0.01lot の必要証拠金が想定超（CFD配分の妥当性）**
   - 実測：CFD口座で **0.01lot=33,661円**（金 ask≈4,147.92）。SETUP_STATUS の見直し閾値「3.2万円/0.01lot を大きく超えるなら配分見直し」の閾値をやや超過（計画時の想定2.9万も超）。
   - 影響試算：`PB GOLD(0.01)+SCA Boost(0.02)=0.03lot 同時`で必要証拠金 ≈**101,000円** → 12万円配分では維持率 **約119%**（計画の138%より薄い）。100%は割らないため即時の危険はないが、金の急変時に余力が薄くなる。
   - 対応候補（ユーザー判断）：GOLD側の同時保有上限の再確認、または CFD配分の微増。**現在GOLD建玉はゼロ**なので緊急性は低い。

2. **⚠️ XM が 7/6 日中にEA外れ（当日の SCA GBPJPY 参加を取り逃し）**
   - XM の mixlog 初出行は 7/6 **18:11（サーバー時刻）**＝再アタッチ（7/7 00:07 日本時間）直後。7/6 朝の SCA レンジ窓（09-12）には未稼働だったため、OANDA FX が取った **09:15 の SCA GBPJPY を XM は取れていない**（SETUP_STATUS 記載の「原因不明の端末再起動でEA消失」の実害）。
   - 現在は再アタッチ済み・自動売買ON・接続OKで正常稼働。**再発監視のため、端末/PC再起動後は「EA搭載・自動売買ON」を必ず確認**（mixlog の DAILY 行 or 起動ログで判定）。

**情報（記録のみ・異常ではない）:**
- OANDA の DAILY 行が `equity=0` で出力（起動直後・口座データ取得前のプレースホルダと思われる）。XM の DAILY は equity=130,000 と正常。ログの実害なし。
- OANDA FX の入金履歴に +100,000 が2件（計20万）記録される一方 balance は 8万 → CFD口座へ12万を内部振替した資金操作の反映。CFD側は入金 deal が履歴窓に出ないが balance 12万で計画どおり。会計上の表示差で、取引・残高に問題なし。
