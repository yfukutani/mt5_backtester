# F_test_setting — フォワードテスト初期（10万円・動作確認）設定ガイド

**作成日:** 2026.07.05
**対象:** MIX_EA.mq5（XM）/ MIX_EA_OANDA.mq5（OANDA）v1.1（運用ログ搭載版）
**目的:** 本格資金投入の前に、**10万円程度の小資金で「全枠が設計通りに動くこと」を確認**し、
以後の改善サイクルに必要な運用ログを取得開始する。**収益性の検証ではない**（それは
バックテスト済み。ここでは機能・執行・ログの検証を行う）。

---

## 1. 運用ログシステム（v1.1で搭載・フォワード改善サイクルの土台）

現状把握: 従来のEquityLogFile等は**バックテスト専用**（OnTester内）でライブでは何も
出力されない。そこでライブ用の運用ログを実装した（`EnableOpsLog`でON/OFF）。

**出力先:** 端末データフォルダの `MQL5\Files\<OpsLogPrefix>_YYYYMM.csv`（月次自動ローテーション・追記式）

**レコード3種（固定11列: time,type,magic,symbol,f1..f6,note）:**

| type | 内容 | 列の意味 |
|---|---|---|
| `DEAL` | 全約定（エントリー/決済） | f1=方向(±1) f2=ロット f3=約定価格 f4=SL f5=TP f6=損益(swap/手数料込) note=IN/OUT |
| `SCA_RANGE` | SCA枠の日次レンジ確定 | f1=高値 f2=安値 f3=幅 f4=D1ATR f5=ドリフト f6=スキップ(0/1) note=ACTIVE/SKIP |
| `DAILY` | 日次スナップショット | f1=equity f2=balance f3=使用証拠金 f4=保有ポジション数 |

**これで可能になる改善分析:**
- 取引をMagic（枠）別に自動集計 → バックテストの同期間結果と枠単位で照合（乖離検知）
- SCA_RANGEでレンジ・スキップ判定・ドリフト（Boost条件）の実測記録 → 「取引しなかった日」
  も含めた完全な意思決定監査
- DEALの約定価格 vs チャート → スリッページ・スプレッド実費の実測
- 分析は既存基盤（`ml/`のPython群）がそのまま読める形式

**動作確認済み:** テスターで月次ファイル生成・3種レコード・Carry複利ロット・SCAのSKIP/
ACTIVE判定・ドリフト記録まで実測確認済み（2026.07.05）。

## 2. 10万円テストの推奨設定

### XM版（MIX_EA）

| 入力 | 値 | 理由 |
|---|---|---|
| `MasterEnable` | true | — |
| `GlobalLotMult` | **1.0** | 全枠が最小ロット0.01で動く（これ以上は下げられない＝10万円が実質最小テスト資金） |
| `Mult_*`（全枠） | **1.0（既定のまま）** | 増レバ配分は本番移行時に設定 |
| `RefCap_PB_USDJPY` | **100000** | risk2%を10万円基準に固定（0だと口座equity連動で意図ずれ） |
| `RefCap_PB_GBPJPY` | **100000** | 同上 |
| `RefCap_CARRY` | **100000** | Carry複利を10万円基準に（0.05ロット相当） |
| `En_*`（全枠） | true（既定） | 全機能の動作確認が目的のため全部ON |
| **`EnableOpsLog`** | **true** | 本テストの主目的の一つ |
| `OpsLogPrefix` | "mixlog" | — |

### OANDA版（MIX_EA_OANDA）

XM版と同一方針。追加考慮:
- `En_SCA_GOLD` / `En_PB_GOLD`: **口座にXAUUSD（商品CFD）アクセスが無い場合はfalse**
- `En_ETH`: false（既定・取扱なし）
- `OpsLogPrefix`: "mixlog_oa"（XMと区別）

### 参考値（この構成のバックテスト実績・決済ベース合算2016-2026）

- **月平均: +6,302円（対10万で約+6%/月）** / 10年純益+756,223円
- **最大DD: 46,074円（10年間の最悪値・対10万46%）**
  ※テスト実施期間（1-2ヶ月）での現実的な変動は**±1〜2万円**程度。10年最悪値の
  DDに遭遇する確率は低いが、テスト資金は「失っても撤退判断に影響しない額」とすること
- 証拠金ピーク: 全枠同時保有でも3〜4万円程度（余力は十分）

## 3. テスト手順とチェックリスト

**期間: 2〜4週間**（下記チェックが全て埋まるまで。SCAは週次で数取引発生する）

### 起動時（初日）
- [ ] エキスパートログに「MIX_EA v1.0 (XM) 起動 | 有効枠数=14/14」（OANDA版は13/13）
- [ ] 自動売買ON・アルゴリズム取引許可を確認
- [ ] 翌営業日に `MQL5\Files\mixlog_YYYYMM.csv` が生成され、DAILY行が出ていること

### 週次確認（毎週末）
- [ ] DEALレコードとMT5口座履歴の突合（件数・損益一致）
- [ ] SCA_RANGE行が毎営業日3銘柄分出ていること（SKIP日も記録される）
- [ ] SCAのエントリー時刻が9-15時（GOLD）/9-12時（JPY）に収まっている
- [ ] GOLD SCAの決済がすべて20:00までに完了している（20時強制決済）
- [ ] **金曜にGOLD SCAの新規エントリーが無い**（金曜スキップ）
- [ ] リバーサル型Boost: SCA_RANGEのドリフト符号とブレイク方向が逆の取引で**ロットが2倍
      （0.02）**になっている
- [ ] Carryのロットが約0.05（RefCap 10万基準）
- [ ] スリッページ: DEALのf3（約定価格）とSL/TP設定値の乖離が異常でない

### 異常時の対応
- 発注されない: アルゴリズム取引許可 / 銘柄の気配値表示追加 / 証拠金 / MasterEnable を確認
- 想定外の銘柄・ロット: 即 `MasterEnable=false` → ログ（mixlog+エキスパートログ）を保全して分析
- ログが出ない: `EnableOpsLog=true` か、Filesフォルダの書込権限を確認

## 4. 合格基準と本番移行

**合格基準（すべて満たすこと）:**
1. チェックリスト全項目クリア・致命的挙動（想定外銘柄/ロット/時刻）ゼロ
2. 各SCA枠で1取引以上・既存枠で計5取引以上の実績
3. DEALログとバックテスト同期間の枠別成績に説明不能な乖離がない
   （方向・時刻・SL/TP構造の一致。損益は相場差で当然異なる）

**本番移行（合格後）:**
1. 資金を50万円に増額（または80万円）
2. 入力を [MIX_EA_UM.md](MIX_EA_UM.md) のシナリオA（50万: 第5案Mult表+SCAミックスB）へ変更
3. **`GlobalLotMult=0.7`で開始**し、1-2ヶ月後に1.0へ（全ブック共通規律）
4. `EnableOpsLog=true` は本番でも常時ON（改善サイクルの生命線）
5. 月次でログを回収し、`ml/`分析基盤で枠別レビュー（バックテスト乖離・執行品質）

## 5. 改善サイクルの回し方（フォワード開始後）

```
月次: mixlog CSVを回収
  → 枠別に集計（Magic単位の純益/勝率/取引数）
  → バックテスト同期間（configs/の該当構成で再実行）と突合
  → 乖離が大きい枠を特定（スプレッド?執行?相場レジーム?）
  → SCA_RANGEログでスキップ率・ドリフト分布・Boost発動率を確認
  → 改善仮説はバックログ方式（every_tick検証→IS/OOSゲート）で検証してから反映
```
