# BfxRev_EA ライブ設置手順（運用PC用）

Bitfinexデレバレッジ・リバウンド（第3/4バックログBF・**2026-07-12採用**。
IS取引数8の留保はユーザー承認済み「取引は少ないが採用」）。
FundingRev v1.1/1.2と同じ「EA単体で完結」設計。

## 1. WebRequestの許可（一度だけ）

MT5（XM）: ツール → オプション → エキスパートアドバイザ → 許可URLに追加:
`https://api-pub.bitfinex.com`
（FundingRevの `https://fapi.binance.com` と併記でよい）

## 2. 設置

1. `experts/BfxRev_EA.mq5` をコンパイル（0エラー確認済み）
2. **BTCUSD D1** チャートにアタッチ（FundingRevとは**別チャート**・Magic 20260724で独立）
3. 起動ログ確認: `BfxRev v1.0 起動 | データ NNNN日 | 急減閾値-10%/5日 | 保有10日 | 災害SL75% | ライブ(API自動更新)`

## 3. 入力（既定のまま）

| 入力 | 値 | 備考 |
|---|---|---|
| DropPct / LookbackDays / HoldDays | 10 / 5 / 10 | 事前登録値。**変更しない**（プラトー: drop15/20・hold5全プラス確認済み） |
| LotSize | 0.01 | 10万円テスト水準 |
| DisasterSL_Pct | 75.0 | MAE最悪-47.5%×1.5・歴史上不発火。**この戦略は日中-25%級の逆行が正常範囲**（ナイフキャッチ型）のためSLを浅くしないこと |
| UseWebRequest | true | Bitfinex API自動取得（1時間リトライ・キャッシュ/CSVフォールバック・全滅時は新規のみ停止） |
| MagicNumber | 20260724 | 他EAと重複しない |

## 4. 動作仕様

- 新D1バーで「前日までのロング建玉が5日で-10%超」ならロング、10バー後に成行クローズ
- **決済はデータ非依存**（保有日数のみ）＝API障害でも必ず閉じる
- 頻度: 約6回/年（58取引/9.5年）。2022年のようにイベントゼロの年もある
- G1（FundingRev）と併走前提の設計（重複7%・相関+0.156・合算効率+39%）
