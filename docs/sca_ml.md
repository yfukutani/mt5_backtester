# SCA_ML — M1機械学習確率システム（GOLD/USDJPY/GBPJPY）

**作成日:** 2026.07.04
**依頼仕様:** 「1分足のローソク足を30本みて、次の10本の間にTPラインに到達している確率を計算するシステム」
**成果物:** データ抽出→学習→較正検証→MQL5組み込みの完全パイプライン + スタンドアロンEA + SCA_EAへの統合フィルター
**結論（正直）:** **確率計算システムは完成し較正も良好**（予測確率帯と実測到達率が一致）。しかし
**収益化は4アプローチすべてで失敗**。モデルの弱い方向シグナル（AUC≈0.52）はスプレッド実費と
「外れ時の逆行」を超えられない。SCA_EA本番はMLフィルター**OFF**のまま（v1.0構成）でフォワード継続を推奨。

## システム構成（コミット済み・再現手順）

```
experts/DataExport.mq5      テスター経由でM1ヒストリーをCSV化（マルチ銘柄対応 v1.10）
ml/train.py                 特徴量生成→ロジスティック回帰学習→AUC/較正表→MQL5ヘッダ自動生成
ml/export_chunks.py         GOLD/GBPJPY全期間回収（年次マイクロテスト×11本）※下記制約の回避
ml/merge_chunks.py          年次チャンクの結合・重複除去・欠落チェック
ml/sim.py                   EA同一ロジックの理論P&Lシミュレーター（学習込み・閾値/SLスイープ）
experts/ml_model_*.mqh      学習済み係数（train.py自動生成、3銘柄）
experts/SCA_ML_EA.mq5       スタンドアロン確率取引EA（M1チャート、Magic 20261100）
experts/SCA_EA.mq5 v1.1     ORBエントリーへのML確認フィルター（UseMLFilter、デフォルトOFF）
```

- **モデル:** L2ロジスティック回帰（自前実装・pandas/numpyのみ）。18特徴量（ATR正規化モメンタム
  ×5、ローソク形状×5、レンジ位置/幅、実現ボラ比、陽線比率×2、tickvol比、時刻sin/cos）。
- **ラベル:** 次10本のhighがclose+2×ATR14(M1)に到達（ロング側。ショートは対称化して1モデルに統合）。
- **分割:** 時系列（train <2024 / valid 2024 / test 2025-2026.06）。**headline数値は全てtest期間**。
- **EA組み込み:** 標準化を係数に畳み込み（w_raw=w/sd, b_raw=b−Σw·mu/sd）、`ML_W_<SYM>[]`として
  ヘッダ出力→コンパイル時埋め込み。特徴量計算はMQL5側でtrain.pyと厳密一致（python t = MQL shift 1）。

## 学習結果（test期間=2025.01-2026.06、学習外）

| 銘柄 | test AUC | ベース率 | P0.4-0.5帯→実測 | P0.5-0.6帯→実測 |
|---|---|---|---|---|
| GOLD | 0.5225 | 33.2% | 43.0% (10,744) | 55.4% (112) |
| GBPJPY | 0.5287 | 33.2% | 43.8% (11,255) | 52.1% (211) |
| USDJPY | 0.5261 | 35.4% | 44.6% (23,606) | 51.7% (822) |

→ **較正は本物**（高い予測帯ほど実測到達率が高い）。ただし0.5超の高確率帯は希少（月数件〜数十件規模）。

## 収益化の検証（4アプローチ・全て失敗）

| # | アプローチ | 結果 | 死因 |
|---|---|---|---|
| 1 | スタンドアロン取引（th0.45/SL4A/TO10本） | EA実測 -55,866円/PF0.55（sim -40,795/PF0.65と一致＝実装正） | **モデルは方向でなくボラを予測**。高確率バー=直前に大きく動いたバーで、TP到達45%は実現するが外れ時の逆行が-1〜-3.5ATRと巨大 |
| 2 | 災害SL除去（ラベル完全整合） | PF 0.65→0.63に**悪化** | タイムアウト裾が-2.7〜-3.5Aへ拡大（ボラ選択の帰結） |
| 3 | 方向純度フィルター（pL高∧pS低） | th0.42/cth0.35: PF1.17でプラス転換も**50取引/1.5年** | 質は上がるが量が実用に満たない（月2.8件・+3千円規模） |
| 4 | ファーストパッセージ再学習（±2A先着ラベル） | 0.5超シグナルほぼ消滅（th0.48で20件・較正崩壊） | 線形モデルの方向予測力ではコスト込みプラスの対称ブラケットに届かない |
| 5 | SCA_EA(ORB)の確認フィルター | th0.34: ベース同等 / 0.36: 純益-19%かつDD悪化 / 0.38: マイナス / 0.40: 28件のみ | ORBブレイク時は既に高ボラ・ロンドン時間＝「ボラ+時間帯」モデルに追加情報なし。閾値応答が非単調=無相関 |

**ベースライン（GOLD ORB・フィルターOFF）: +134,501円/PF1.44/DD16.1% — これを上回る構成なし。**

## 技術ノート（再現・運用時の注意）

- **GOLD単独チャートのバックテストが異常終了する**既知の機械問題（45秒でtester forced to close）
  → DataExportはUSDJPYチャートで実行し`ExportSymbols`で他銘柄を書き出す方式で回避。
- **参照銘柄のヒストリーは「テスト開始前年の1月1日」からしか事前ロードされない** + テスト進行と
  同期したM1バー生成が非常に重い（10年で1時間超）→ `export_chunks.py`の年次マイクロテスト
  （期間数日×11本、テスト開始前データもCopyRatesで取得可能な性質を利用）で全期間を数分×11で回収。
- **MetaTrader5 Python APIは不可**: 端末ヒストリーは直近数ヶ月のみ・maxbars=100000制限。
- 学習データCSV（各200MB前後）はリポジトリに含めない（scratchpad/ml/に生成）。
- 特徴量・ラベル定義を変える場合はtrain.py/sim.py/SCA_ML_EA.mq5/SCA_EA.mq5の4箇所を必ず同期すること。
